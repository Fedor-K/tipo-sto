# TIPO-STO: Техническое задание

## Проблема рынка

### Текущая ситуация

На рынке CRM для автосервисов существует разрыв:

| Тип решения | Примеры | Плюсы | Минусы |
|-------------|---------|-------|--------|
| **Специализированные CRM** | STOCRM, YCLIENTS, iDirector | Простой интерфейс, быстрый старт | Нет интеграции с 1С |
| **Решения на базе 1С** | 1С:Автосервис, Альфа-Авто | Полный учёт, бухгалтерия | Сложный интерфейс, дорогое внедрение |

### Главная боль владельцев СТО

**Сложность существующих систем:**

- 1С — мощная, но слишком сложная для ежедневной работы
- Владельцы не хотят и не могут тратить время на настройку CRM
- Простые CRM не связаны с 1С → приходится вводить данные дважды
- Сотрудники саботируют сложные системы

### Статистика

- Только 10% автосервисов в России прибыльны
- 70% внедрений CRM заканчиваются провалом
- Около 76 000 независимых автосервисов в России

## Концепция TIPO-STO

### Миссия

> **Простая CRM для ежедневной работы с автоматической синхронизацией с 1С**

### Принцип работы

```
┌─────────────────────────────────────────┐
│         TIPO-STO (простой UI)           │
│  • Запись клиента — 3 клика            │
│  • Заказ-наряд — 1 экран               │
│  • Закрытие заказа — кнопка            │
└─────────────────┬───────────────────────┘
                  │ автоматически
                  ▼
┌─────────────────────────────────────────┐
│              1С (бухгалтерия)           │
│  • Документы                            │
│  • Налоги                               │
│  • Отчётность                           │
└─────────────────────────────────────────┘
```

### Ключевые принципы

1. **Работает сразу** — минимум настроек, подключил 1С и работай
2. **Простой интерфейс** — приёмщик обучается за 10 минут
3. **Нет двойного ввода** — всё автоматически уходит в 1С
4. **Мобильность** — работает с телефона
5. **Облачный** — не нужен IT-специалист

## Функциональные требования

### MVP (первая версия)

#### 1. Запись клиентов
- Календарь записи по постам/подъёмникам
- Быстрое создание записи (клиент + авто + услуга)
- Просмотр загрузки на день/неделю

#### 2. Заказ-наряды
- Создание заказ-наряда из записи
- Добавление работ (из справочника 1С)
- Добавление запчастей (из справочника 1С)
- Статусы: Новый → В работе → Готов → Выдан
- Печать заказ-наряда

#### 3. База клиентов и автомобилей
- Карточка клиента (ФИО, телефон, email)
- Карточка автомобиля (марка, модель, VIN, госномер)
- История обслуживания

#### 4. Интеграция с 1С
- Синхронизация справочника номенклатуры (работы, запчасти)
- Синхронизация контрагентов
- Передача закрытых заказ-нарядов в 1С
- Получение остатков склада

### Версия 2.0 (развитие)

- Telegram-бот для клиентов (запись, статус ремонта)
- SMS/WhatsApp напоминания
- Онлайн-оплата
- Программа лояльности
- Аналитика и дашборды
- Мобильное приложение для клиентов

## Техническая архитектура

### Стек технологий

| Компонент | Технология |
|-----------|------------|
| Backend API | Python + FastAPI |
| База данных | PostgreSQL |
| Кэш/очереди | Redis |
| Веб-интерфейс | React или Vue.js |
| 1С интеграция | HTTP-сервисы 1С / OData |
| Хостинг | VPS (Windows Server для 1С) |

### Схема интеграции с 1С

```
┌─────────────┐     HTTP/REST      ┌─────────────┐
│  TIPO-STO   │◄──────────────────►│  1С:Сервер  │
│   Backend   │                    │  (HTTP-     │
│  (FastAPI)  │                    │  сервисы)   │
└─────────────┘                    └─────────────┘
       │                                  │
       ▼                                  ▼
┌─────────────┐                    ┌─────────────┐
│ PostgreSQL  │                    │   База 1С   │
└─────────────┘                    └─────────────┘
```

### Синхронизация данных

| Объект | Направление | Частота |
|--------|-------------|---------|
| Номенклатура | 1С → CRM | Раз в час |
| Остатки склада | 1С → CRM | По запросу |
| Контрагенты | Двусторонняя | При изменении |
| Заказ-наряды | CRM → 1С | При закрытии |

## Интерфейс

### Основные экраны

1. **Главный экран** — календарь записей на сегодня
2. **Запись** — форма создания записи
3. **Заказ-наряд** — работа с заказом
4. **Клиенты** — поиск и карточки клиентов
5. **Настройки** — подключение 1С

### Требования к UX

- Максимум 3 клика до любого действия
- Крупные кнопки (работа на планшете)
- Минимум полей для заполнения
- Автоподстановка данных
- Работа оффлайн с последующей синхронизацией

---

## Конкурентные преимущества

1. **Простота** — не нужен специалист для настройки
2. **Интеграция с 1С** — нет двойного ввода данных
3. **Своё размещение** — данные на вашем сервере
4. **Открытый код** — возможность доработки
